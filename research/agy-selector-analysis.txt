## TL;DR

**agy v1.1.1 implements a two-layer task routing architecture**: (1) **Execution-mode selection** based on prompt analysis (plan vs accept-edits), and (2) **LLM-driven tool/model selection** with schema-validated output gates. Unlike kimi-delegate's "load all, let LLM choose" approach, agy pre-gates execution modes then exposes full toolset within that mode. The core mechanisms worth borrowing are: mode-gating (plan/accept-edits branching), prompt templating via frontmatter (schema + model override per task), git-worktree isolation per agent, and salvage-mode schema validation.

## BORROWABLE MECHANISMS (3–5 for Claude Code + kimi-delegate)

### 1. **Mode-Gating via Prompt Analysis**
   - **What agy does**: Pre-classify prompt into `plan` vs `accept-edits` → load corresponding system prompt template
   - **How Claude Code can use it**: Instead of loading all skills, analyze the task description and pre-gate to: `refactoring` (vs `new-feature` vs `bugfix` vs `docs`) → load subset of skills + model tier (Flash for simple, Pro for complex)
   - **Benefit**: Reduce cognitive load on LLM (fewer irrelevant tools), lower latency
   - **Implementation**: Lightweight classifier in kimi-delegate (5–10 line regex or heuristic; OR LLM-based: quick categorization call before main work)

### 2. **Frontmatter-Based Overrides (Per-Task Metadata)**
   - **What agy does**: YAML frontmatter in brief files: `model:`, `timeout:`, `schema:`
   - **How Claude Code can use it**: Extend brief format with `retry-budget:`, `max-parallel:`, `skill-filter:` → allow humans to override system defaults per decomposed subtask
   - **Benefit**: Humans provide domain knowledge (e.g., "use Claude for this, Gemini for that") without modifying core router
   - **Implementation**: Parse frontmatter in kimi-delegate skill briefs; pass overrides to subagent launch

### 3. **Salvage-Mode Schema Validation**
   - **What agy does**: On schema validation failure, return partial JSON with `_missing` and `_invalid` fields + `PARTIAL(schema)` status (reviewable, not failed)
   - **How Claude Code can use it**: Instead of binary pass/fail on artifact checks (tests, lint), return structured feedback with **what succeeded, what failed, what was skipped** → let human judge if retry is needed
   - **Benefit**: Reduce agent thrashing on edge cases; transparency on partial progress
   - **Implementation**: Extend artifact validation scripts to emit JSON with structured diagnostics (not just "FAILED")

### 4. **Git Worktree Isolation + Atomic Diffs**
   - **What agy does**: One worktree per agent (branch `agy/<name>`); final diff via `git diff base..agy/<name>`; merge is human decision
   - **How Claude Code can use it**: Adopt same pattern in kimi-delegate (already supports `--no-worktree` for disjoint scopes); ensure every agent's output is reviewable as a clean diff before merge
   - **Benefit**: Safe parallelism; rollback on bad merge; human audit trail
   - **Implementation**: Already present in kimi-delegate; reinforce in documentation

### 5. **Timeouts + Quota Detection**
   - **What agy does**: Per-agent `--print-timeout` (Go duration); detect quota exhaustion via log tail regex (`/quota|rate limit|credit|unauthorized|not logged in/i`)
   - **How Claude Code can use it**: Enhance kimi-delegate fallback logic: on agent timeout or detected quota, **swap to next model tier** or **queue for later** instead of immediate retry (exponential backoff)
   - **Benefit**: Graceful degradation under rate limits
   - **Implementation**: Post-execution log scan; conditional retry or escalation
